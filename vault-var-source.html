1.4.3.2.1 The Vault ((var)) source

The vault type supports configuring a Vault server as a ((var)) source.

For a more complete explanation on how to set up Vault, check out the docs for the Vault credential manager. The documentation here is going to be pretty sparse until var_sources is more complete so as to avoid duplication.

Consider this "early access" - we don't expect the following configuration schema to change, but until we fully commit to this direction and have cluster-wide var sources we don't want to shuffle around the docs too much.

The following fields can be set under config:

url: string

Required. The URL of the Vault API.

ca_cert: string

Optional. The PEM encoded contents of a CA certificate to use when connecting to the API.

path_prefix: string

Optional. Default /concourse. A prefix under which to look for all credential values.

See Credential lookup rules for more information.

shared_path: string

Optional. An additional path under which credentials will be looked up.

See Configuring a shared path for more information.

namespace: string

Optional. A Vault namespace to operate under.

client_cert: string

Optional. A PEM encoded client certificate, for use with TLS based auth.

See Using the cert auth backend for more information.

client_key: string

Optional. A PEM encoded client key, for use with TLS based auth.

See Using the cert auth backend for more information.

server_name: string

Optional. The expected name of the server when connecting through TLS.

insecure_skip_verify: boolean

Optional. Skip TLS validation. Not recommended. Don't do it. No really, don't.

client_token: string

Optional. Authenticate via a periodic client token.

See Using a periodic token for more information.

auth_backend: string

Optional. Authenticate using an auth backend, e.g. cert or approle.

See Using the approle auth backend or Using the cert auth backend for more information.

auth_params: string: string

Optional. A key-value map of parameters to pass during authentication.

See Using the approle auth backend for more information.

auth_max_ttl: duration

Optional. Go duration format. Maximum duration to elapse before forcing the client to log in again.

auth_retry_max: duration

Optional. Go duration format, default 5m. When failing to authenticate, give up after this amount of time.

auth_retry_initial: duration

Optional. Go duration format, default 1s. When retrying during authentication, start with this retry interval. The interval will increase exponentially until auth_retry_max is reached.
